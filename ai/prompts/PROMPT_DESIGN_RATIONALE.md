## 1. 设计目标

为 DeepSeek-V3 构建一套完整的 Prompt 系统，实现：

> **输入**：单词列表 + 用户背景  
> **输出**：一段让人"忘不掉"的情感叙事场景，帮助永久记忆单词

核心差异点：不背单词，背**故事**。

---

## 2. 认知心理学原理

### 2.1 情绪记忆效应（McGaugh, 2000）

**原理**：情绪唤起会激活杏仁核-海马体通路，使记忆更持久。

**应用**：
- 场景 Prompt 要求**必须有情感触发点**（惊喜/感动/搞笑等）
- 不是中性叙事，是"被击中"的瞬间
- 情感越强烈，记忆越深刻（可参考血氧仪实验中的情感图片记忆优势）

### 2.2 第二人称沉浸（Self-Reference Effect）

**原理**：信息与自我关联时记忆效果最佳（Rogers et al., 1977）。

**应用**：
- 全文使用第二人称"你"
- 读者代入主角视角，而非旁观者
- 场景中的行为、感受都是"你的"，不是"他的"

### 2.3 感官编码（Multi-Sensory Encoding）

**原理**：多感官编码使记忆更牢固（记忆的多通路理论）。

**应用**：
- 场景 Prompt 强制要求至少 3 种感官描写
- 视觉 + 听觉 + 嗅觉 + 触觉 + 味觉任意组合
- 感官越具体，场景越真实，记忆越牢固

### 2.4 生成效应（Generation Effect）

**原理**：自己生成的内容比被动接收记得更牢（Slamecka & Graf, 1978）。

**应用**：
- 复习提示不直接给答案，而是给"记忆钩子"
- 用户主动回忆 → 记忆更深
- 提示只是推一把，不是替用户做

### 2.5 间隔效应与遗忘曲线（Ebbinghaus）

**原理**：遗忘速度先快后慢，及时复习事半功倍。

**应用**：
- 提示层级渐进（3级），模拟间隔复习
- Level 1 最模糊 → Level 3 最具体，记忆逐渐激活
- 最多提示 4 次，第 5 次才给答案——强迫主动回忆

### 2.6 位置记忆法（Method of Loci）

**原理**：将抽象信息与具体空间位置关联。

**应用**：
- 复习提示暗示单词在原文中的位置（"开头第三句"、"描述颜色的那句"）
- 原场景的三段式结构（起承转合）本身就提供了空间锚点
- 通勤场景（地铁/咖啡厅）有天然的空间感

## 3. 各 Prompt 设计详解

### 3.1 场景生成 Prompt

#### 为什么这样设计 System Prompt

| 设计要素 | 认知心理学依据 | 具体实现 |
|----------|--------------|---------|
| 随机情感类型 | 避免记忆同质化，每次新情绪激活不同记忆网络 | 8种情感类型×8种场景组合 |
| 三段式结构 | 叙事结构增强情节记忆 | 开头（日常感）→ 中段（转折）→ 结尾（情感闭环） |
| 感官细节要求 | 多感官编码 | 至少3种感官描写 |
| 下划线标注 | 明确词汇锚点 | `___word___` 格式，视觉标记 |
| 禁止字眼 | 避免暴露学习目的，保持沉浸感 | "单词""背""记忆""学习"均禁止 |
| 字数上限（200-400） | 手机端阅读体验 + 注意力资源有限 | 认知负荷控制在短时记忆容量内 |

#### 为什么用下划线而非星号/引号

- 下划线视觉冲击力强，像"被标记的重点"
- 不影响阅读流畅度（阅读时大脑自动忽略标记）
- 移动端 UI 容易解析（下划线是标准 Markdown 语法）

#### 为什么不一次性生成多个场景供选择

- 用户决策成本高 → 反而降低学习意愿
- v1.0 聚焦单场景高质量生成
- 评估 Prompt 作为质量门控，保证生成质量

---

### 3.2 场景评估 Prompt

#### 为什么需要评估 Prompt

AI 生成存在随机性，单靠 System Prompt 无法 100% 保证质量。
评估 Prompt 的作用：
1. **生成后自检**：不放过不合格场景
2. **重生成依据**：告诉模型哪里不对，让其自我修正
3. **质量数据积累**：记录评分趋势，优化 Prompt 本身

#### 4个维度的设计逻辑

| 维度 | 为什么重要 | 及格线设计依据 |
|------|----------|--------------|
| 情感强度 | 这是 LinguaFlow 的核心差异，情感不达标 = 产品失效 | ≥7/10（勉强及格）|
| 词汇覆盖率 | 词汇是学习目标，覆盖率低于 95% = 任务失败 | ≥95% |
| 故事流畅度 | 若用户觉得是"例句罗列"，沉浸感归零 | ≥7/10 |
| 个人化程度 | 第二人称+具体场景是记忆效果的保障 | ≥7/10 |

---

### 3.3 复习引导 Prompt

#### 为什么不直接给答案

直接给答案 = 跳过回忆 = 生成效应失效 = 记忆不深。

#### 3级提示的设计逻辑

```
Level 1: 情感共鸣（远距线索）
↓ 用户仍记不住
Level 2: 语境定位（中等线索）
↓ 用户仍记不住
Level 3: 语义暗示（近距线索，但仍让用户猜）
↓ 用户仍记不住
展示答案（放弃主动回忆）
```

这模拟了真实记忆过程：
> "我知道那个词，但一时想不起来" → 朋友给个暗示 → "哦对！就是这个！"

#### 提示数量的限制（最多4次）

防止用户"刷提示"绕过回忆过程。
第5次直接给答案是兜底，确保用户体验不断裂。

#### 为什么提示要随机开头

避免语言模式化，让每次提示都像"真正的朋友在帮你"。
模式化语言会让用户感觉被 AI 敷衍 → 失去信任感。

## 4. 随机性控制策略

### 为什么需要随机性

- 同一组单词多次学习时，若场景完全相同，记忆效果递减
- 用户会觉得"我好像见过这个场景"，降低新鲜感
- 随机情感类型 + 随机场景组合 = 无限变化可能

### 当前随机化方案

| 维度 | 随机池大小 | 效果 |
|------|----------|------|
| 情感类型 | 8种 | 8× 不同情绪基调 |
| 生活场景 | 8种 | 8× 不同环境背景 |
| 提示开头 | 4种 | 每次提示语气不同 |
| 用户背景 | 用户自定义 | 无限组合 |

**理论最大组合数**：8 × 8 × ∞ = 无限变化

### 未来优化方向（v2.0）

- 加入时间维度（早/午/晚/夜 × 不同光线）
- 加入天气维度（晴/雨/雪/雾）
- 加入人物关系（同事/朋友/陌生人/宠物）
- 支持用户自定义情感偏好（"我更喜欢搞笑场景"）

## 5. 风险与对策

| 风险 | 严重程度 | 对策 |
|------|---------|------|
| 单词遗漏（覆盖率<95%）| 高 | 评估 Prompt 兜底，不合格则重生成 |
| 情感平淡 | 中 | 随机情感类型强制避免中性叙事 |
| 暴露学习目的 | 中 | 禁止字眼清单 + 评估 Prompt 检查 |
| 词汇标注错误 | 低 | 统一下划线格式，评估 Prompt 检查格式 |
| 生成超时/失败 | 低 | 降级策略：返回默认场景模板 |
| 提示变成答案 | 低 | Level 3 限制：不透露单词本身，只给语义方向 |
