## 💡 经验教训

**本次 Sprint 0 学到的关键教训（持续积累）：**

1. **Agent 超时风险**：多步骤文档生成任务（PRD）需设置更短的单步超时 + 中间 checkpoint，避免长链条任务彻底超时丢失进度
2. **定时任务防御**：09:50 GLM 抢购提醒依赖 cron 触发，今日因 SIGTERM 信号丢失。后续考虑：在 09:00 搜索阶段主动预埋 50 分钟后的提醒，而非依赖独立 cron（减少单点故障）
3. **飞书群 chat_id 未记录**：无法推送团队消息。经验：首次配置飞书机器人后应立即记录 chat_id 至 MEMORY.md
