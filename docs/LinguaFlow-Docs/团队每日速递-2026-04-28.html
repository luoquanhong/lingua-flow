4️⃣ 经验教训

**Git Push 冲突的标准处理流程（已固化为习惯）：**
1. `git fetch origin master`
2. `git log --oneline origin/master -3` 确认远程状态
3. `git rebase origin/master`
4. `git push origin master`
教训：每次 pull 前务必先 fetch 确认远程是否有新 commit，避免 rebase 时丢失本地未推送的变更。
