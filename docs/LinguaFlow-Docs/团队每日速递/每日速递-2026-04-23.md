## 4. 经验教训

### ✅ 文档中心故障已修复——根源分析
**问题：** `docs/index.html` 中的链接指向 `.md` 文件，但 `convert-docs.js` 生成的 `.html` 文件未同步到 GitHub 仓库。

**根因：** `convert-docs.js` 的 `DOCS_DIR` 路径配置为 `/workspace/LinguaFlow-Docs/`（不在 git 仓库内），导致生成的 HTML 文件放错目录，GitHub Pages 读取不到。

**教训：**
1. `convert-docs.js` 的 `DOCS_DIR` 必须指向 `lingua-flow/docs/LinguaFlow-Docs/`（git 仓库内）
2. 文档变更 Regression Test 需新增：确认 `docs/` 目录返回 HTTP 200（不只检查主页）
3. HTML 文件属于构建产物，需与源文件（.md）一起 git add + push

### ✅ QA 检测流程规范化
发现文档故障后，建立标准 QA 检测步骤：
1. GitHub Pages URL HTTP 200/404 检查
2. `docs/index.html` 链接格式检查（.md vs .html）
3. HTML 文件实际存在性验证
