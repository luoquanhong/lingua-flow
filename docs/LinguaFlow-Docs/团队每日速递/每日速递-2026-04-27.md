## 4. 经验教训

### ✅ GitHub push 被拒后的正确处理流程

**问题：** 首次 push 遇到 `rejected` 错误（远程有新 commit）。

**正确操作（已实践验证）：**
```bash
git pull --rebase origin master   # 先拉取再合并
git push origin master            # 再推送
```
**教训：** 严禁直接 `git push -f`，先 rebase 再推送可保留双方历史。

---

### ⚠️ 竞品早报补做流程（网络故障后）

**触发条件：** 09:00 搜索失败，网络恢复后应立即补做。
